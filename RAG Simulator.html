⚡ RAG Architecture Flow

1

Document Input

Large documents are uploaded into the system.

2

Chunking

Documents are split into smaller chunks like paragraphs or sentences.

3

Embeddings

Each chunk becomes a vector embedding.

4

Vector Database

Embeddings are stored inside a vector DB like Pinecone or Weaviate.

5

User Query

User question is converted into embeddings.

6

Similarity Search

Semantic search retrieves the most relevant chunks.

7

LLM Response

Retrieved chunks are passed to the LLM for final answer generation.
